## Intent

Implement the second advanced IgWeb routing slice: **explicit `resource` sugar**.

Resource sugar reduces repeated REST-shaped route boilerplate while preserving the central IgWeb rule:
everything lowers to ordinary flat `.igweb` route semantics and then to inspectable `.ig`.

The action table is a **validator, not a generator**:

- contracts are always named explicitly;
- methods/suffixes are checked against a small closed table;
- no controller conventions;
- no auto contract naming;
- no pluralization/inflection;
- no hidden parent loading.

## Resource Grammar

Add only:

```igweb
resource <name> "<base>" {
  index  GET             -> TodoIndex
  create POST            -> TodoCreate requires idempotency
  show   GET    "/:id"   -> TodoShow
  update PATCH  "/:id"   -> TodoUpdate requires idempotency
  delete DELETE "/:id"   -> TodoDelete requires idempotency

  member     POST "/:id/done" -> TodoDone requires idempotency
  collection GET  "/search"   -> TodoSearch
}
```

Rules:

- `resource` may appear only inside `app { ... }` or inside `scope { ... }`.
- `<name>` is author-facing only; it is not used for contract-name derivation.
- `<base>` is a quoted path starting with `/`.
- routes generated by resource compose with the active P16 scope prefix.
- `resource` disappears before existing route grouping / `.ig` generation.
- line-positioned `IgwebError` for malformed resource lines and unclosed resources.

## Closed Action Table

Use a closed table as a validator:

| action | method rule | suffix rule | generated suffix |
|---|---|---|---|
| `index` | must be `GET` | no suffix allowed | `/` |
| `create` | must be `POST` | no suffix allowed | `/` |
| `show` | must be `GET` | suffix optional; default `/:id` | supplied or `/:id` |
| `update` | must be `PATCH` or `PUT` | suffix optional; default `/:id` | supplied or `/:id` |
| `delete` | must be `DELETE` | suffix optional; default `/:id` | supplied or `/:id` |
| `member` | explicit method required | quoted suffix required | supplied suffix |
| `collection` | explicit method required | quoted suffix required | supplied suffix |

Important: this table validates shape; it must not invent contract names. For example,
`index GET -> TodoIndex` is valid because `TodoIndex` is explicitly authored.

If the table above feels too large after verify-first, reduce the implementation to
`index/create/show/member/collection`, but document the tradeoff and keep it internally consistent.

## Required Lowering

Resource expands to flat route entries before the existing P4/P16 route pipeline.

Example:

```igweb
resource todos "/todos" {
  index  GET            -> TodoIndex
  create POST           -> TodoCreate requires idempotency
  show   GET    "/:id"  -> TodoShow
  member POST "/:id/done" -> TodoDone requires idempotency
}
```

must behave as if authored:

```igweb
route GET  "/todos"          -> TodoIndex
route POST "/todos"          -> TodoCreate requires idempotency
route GET  "/todos/:id"      -> TodoShow
route POST "/todos/:id/done" -> TodoDone requires idempotency
```

Then the existing lowerer handles regexp/capture, idempotency guards, grouping, 404, and 405.

## Required Behavior

### Explicit contracts only

Valid:

```igweb
index GET -> TodoIndex
```

Invalid / out of scope:

```igweb
index GET
```

No generated `TodoIndex`, no `TodosController#index`, no pluralization, no convention fallback.

### Same-path grouping and 405

`index GET -> TodoIndex` and `create POST -> TodoCreate requires idempotency` share the same composed
path. They must land in the same generated pattern group so:

- `GET /todos` calls `TodoIndex`;
- `POST /todos` calls `TodoCreate` or 400 if keyless;
- `DELETE /todos` returns 405, not 404.

### Scope composition

Resource must compose with P16:

```igweb
scope "/accounts/:account_id" {
  resource todos "/todos" {
    index GET -> AccountTodosIndex
    show  GET "/:todo_id" -> AccountTodoShow
  }
}
```

Generates equivalent flat paths:

```igweb
route GET "/accounts/:account_id/todos"          -> AccountTodosIndex
route GET "/accounts/:account_id/todos/:todo_id" -> AccountTodoShow
```

### Duplicate params

P16 duplicate-param refusal still applies after resource+scope composition. For example:

```igweb
scope "/todos/:id" {
  resource comments "/comments" {
    show GET "/:id" -> CommentShow
  }
}
```

must refuse duplicate `:id`.

## Closing Report (2026-06-19)

**Deliverable:** `resource <name> "<base>" { … }` implemented in `lang/igniter-compiler/src/igweb.rs` as
authoring sugar that validates REST action lines against a **closed table** and expands them to flat
routes, **reusing the P16/P4 route path verbatim** (zero new lowering). Proof doc:
`lab-docs/lang/lab-igniter-web-routing-resource-sugar-p17-v0.md`.

**Implementation (only `igweb.rs` + its tests changed):**
- `lower_igweb` tracks `in_resource: Option<(String, usize)>` (absolute base = scope + resource base,
  plus header line for unclosed-block diagnostics); `}` closes the innermost block resource → scope →
  app; resource-body lines dispatch to `parse_resource_action`.
- `parse_resource_header` parses `resource <name> "<base>" {` (name author-facing only, never derived).
- `resource_action_suffix` = the closed table (index/create/show/update/delete/member/collection),
  validating method + suffix and returning the effective suffix; never invents a name/method/path.
- `parse_resource_action` splits on `->` (clear missing-contract error), validates, then **synthesizes
  the equivalent flat `route` tail and delegates to `parse_route`** — so composition, duplicate-param
  refusal, the idempotency 400 guard, same-path grouping, and 404/405 all come from the existing path.
- Unclosed resource reported.

**Anti-magic upheld:** explicit contracts only (missing `->` refused); no auto-naming/pluralization; no
hidden parent loading; no resource-level idempotency default; same-path grouping gives 405 for free.

**Proof — all green:**
- `cargo test --lib igweb::tests` → **29 passed** (16 prior + 13 new).
- `cargo test --test igweb_lowering_tests` → **4 passed** (3 prior + 1 new). The new

  **byte-identically** to the flat form **and** compiles clean through the real multifile compiler

**Closed surfaces honored:** no `nested`/`via`, no source-map, no runner/CLI/server change, no canon, no
