# Igniter Design System Lab Prototype

This package contains lab-only brand, syntax, and product-interface design assets for Igniter Lang experiments.

It is a frontier design-system snapshot, not a canonical public brand package, stable UI framework, production website, or release asset. The HTML files are preserved as inspectable design artifacts and implementation references for lab tools such as the IDE, view engine, GUI engine, and future docs mockups.

## Contents

- `assets/ig-brand.css` - shared color, typography, and primitive CSS tokens.
- `assets/ig-mark.js` - dependency-free SVG mark renderer for prototype pages.
- `assets/design-canvas.jsx` - design-canvas helper used by generated mockup artifacts.
- `Igniter-Lang Brand System.html` - brand reference snapshot.
- `Igniter-Lang Syntax.html` - syntax theme and token-color reference snapshot.
- `Igniter IDE.html` - IDE shell visual reference snapshot.
- `Igniter Design System Plan.html` - implementation brief for applying the visual system to lab surfaces.
- `Igniter-Lang README.html`, `Igniter-Lang Landing.html`, and `Igniter-Lang Final Candidates.html` - historical/public-facing mockup snapshots, not published product claims.
- `assets/*.png` - syntax-check visual specimens.

## Transfer Status

This package belongs to the `igniter-lab` repository as lab/frontier evidence. It may inform future product, docs, or IDE work, but it does not create public brand authority, stable API/UI guarantees, production readiness, release evidence, or canonical language documentation.

## Hygiene

Generated output, local screenshots, local state, logs, and build artifacts should stay out of git. Keep absolute local paths and local file URL links out of committed docs and artifacts.
