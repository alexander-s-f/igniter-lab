-- settlement.ig — history-aware ledger contract
module SparkCRM.Ledger
import Lang.Core.Money

contract Settlement {
  input   account_id: String
  input   base_bid:   Decimal[2]
  input   tax_rate:   Decimal[4]

  read    ledger: History[Decimal[2]] from TBackend lifecycle: :durable

  compute gross_bid = mul(base_bid, tax_rate)
  compute prior     = ledger.as_of(clock.now)
  compute delta     = gross_bid - prior

  invariant non_negative predicate: gross_bid >= 0.00 severity: :error
  snapshot point = gross_bid at: clock.tick

  output  gross_bid: Decimal[2]
  output  delta:     Decimal[2]
}

§03

Meaning-Bearing Colors

preserved · retuned for ink
core
Pure compute & core fragments. The trustworthy path.
#5DB87A · was #22c55e
escape
Side effects & escape hatches. Handle with care.
#F0A868 · = brand Amber
temporal
Time, history & clocks. The one cool accent — earns its contrast.
#5EC8D8 · was #06b6d4
oof
Errors & failed invariants. Aligned to brand's signal red.
#D9694A · was #ef4444

These four colors are not decoration — they classify fragments in the language (a compute is core, an escape is privileged, a temporal read touches history). We keep all four meanings, but warm them so they sit on ink instead of cold gray. escape simply is brand Amber; temporal cyan is deliberately the only cool note in the system — time should feel different from everything else.
