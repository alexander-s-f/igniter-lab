DS-IDE-01 Design System Re-skin No layout changes

Apply Ember-on-Ink to Igniter IDE

Migrate the IDE's cold gray/blue chrome to the Ember-on-Ink brand without touching layout, component structure, or behavior. This is a token-and-theme swap: warm the surfaces, move the accent from blue to ignition orange, swap the Monaco theme, and preserve the four meaning-bearing semantic colors (retuned for warm ink). Stack is SvelteKit + Tailwind v4 + Tauri.

Primary accentblue-700 → #FF6A3D
Groundgray-950 → #15110D
App markoval огонёк (sub-product)
Risklow · visual only

⏃

Preserve the meaning-bearing colors

The IDE's core / escape / temporal / oof colors aren't theme decoration — they classify fragments in the language (a compute is core, an escape is privileged, a temporal read touches history, a failed invariant is oof). Keep all four meanings. Only shift their hue/lightness so they harmonize on warm ink. Never collapse them into a single accent.

core · pure escape · effectful temporal · time oof · error

§06

Definition of Done

acceptance criteria
01

No blue-* or raw gray-* utilities remain in src/** (semantic core/escape/temporal/oof classes excepted).

02

Monaco editor renders the igniter-dark Ember-on-Ink theme; cursor and selection are ignite; .ig tokens match the Syntax spec.

03

The four semantic colors still carry meaning and remain visually distinct — verify on a DAG with a core compute, an escape node, a temporal read, and a failed invariant.

04

The oval mark appears in the titlebar and status bar; the favicon is the original mark.

05

Zero layout / behavior diffs — panel sizes, drag handles, keyboard shortcuts, and component structure are untouched. Pure visual PR.

06

Contrast holds: body text ≥ AA on ink; the IDE mock in Igniter IDE.html is the visual source of truth.
