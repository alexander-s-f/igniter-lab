◆  THE WHOLE IDEA

Three promises, kept by the compiler.

Everything in igniter-lang comes back to these. They're not features you turn on — they're the shape of the language.

01 / declared

Declared, not discovered

Every input and output is named up front. If a value isn't declared, it can't enter the program — so there's nothing to reverse-engineer later.

02 / observable

Observable by default

Each compute is a node in a small dependency graph. You can watch any value resolve — no black box, no stepping blind through someone else's effects.

03 / time-aware

Time-aware from the start

History and temporal types are first-class. "What did this value look like last Tuesday?" is a real question with a real, provable answer — built in, not bolted on.

◆  ANATOMY OF A CONTRACT

Five words you'll use all day.

A contract is the unit of work. Read one and you've read the language.

bid_summary.ig
module SparkCRM.Finance

contract BidSummary {
  input   base_bid: Decimal[2]
  input   tax_rate: Decimal[4]

  compute gross_bid = mul(base_bid, tax_rate)

  output  gross_bid: Decimal[2]
}
contract

The unit of work

A named, self-contained block. It owns its inputs, its logic, and its outputs — nothing leaks across the boundary.

input

What goes in

Declared parameters with explicit types. The compiler refuses anything undeclared, so the surface area is always exactly what you see.

Decimal[2]

Precision that's part of the type

Money keeps two places, tax rates keep four. Rounding is a decision you make on purpose, never an accident the runtime makes for you.

compute

The observable step

A single derivation, named so it can be watched, cached, and proven. Chain them and you've drawn a dependency graph.

output

What comes out

The promised result, re-typed at the boundary. If the computation can't satisfy it, the program never compiles.

◆  COMPILATION PRODUCES EVIDENCE

Every result arrives with a receipt.

When igniter-lang compiles a contract, it doesn't just emit a binary — it emits a sealed .igapp with the proof of what happened inside. Inputs, computes, types, and effects, all witnessed.

Hand the receipt to a teammate, a CI job, or an auditor six months from now. The answer is the same, and it's checkable — not "trust me," but "here's the proof."

◆  WHAT'S IN THE BOX

Small surface, serious guarantees.

Typed contracts

Inputs and outputs are typed and checked before anything runs. Decimal precision is part of the type.

Dependency graph

Computes form a DAG you can inspect, cache, and replay deterministically.

History & BiHistory

First-class temporal types. Ask what a value was, not just what it is.

Sealed artifacts

Compile to a proof-carrying .igapp — the evidence travels with the result.

Clear diagnostics

Errors point at the contract, name the rule, and tell you what to fix. No stack-trace archaeology.

Ruby-native CLI

Ships as a gem. igc compile fits the toolchain you already have.
