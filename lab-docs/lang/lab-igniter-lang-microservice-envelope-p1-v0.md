## Scope

This document defines the Igniter microservice envelope for an `igniter-lang`
service that executes real IO through CapabilityExecutors. It answers all six
card questions with citations to `igniter-lang` canon and closed `igniter-lab`
evidence.

**Authority boundary:** `igniter-lang` canon and experiment-pass lab evidence
only. The old Ruby `igniter` gem, Rack gem core, ActiveRecord, ORM, and
`Igniter::ContractBuilder` are **not authority** for any finding here.

**Not a server implementation.** No accept-loop code, no Rack middleware chain,
no production runtime claim. All shapes are design proposals grounded in
evidence already established by P1 and prior lab cards.

---

## Upstream Route (from P1)

P1 established the canonical IO Runtime route:

```text
effect contract
  -> capability/effect_binding            (experiment-pass: PROP-035 subset)
  -> SemanticIR ESCAPE + requirements.json
  -> RuntimeMachine load/evaluate         (proven: Stage 1/2)
  -> CapabilityExecutor registry          (GAP — LANG-IO-CAPABILITY-EXECUTOR-P1)
  -> typed receipt / failure / unknown_external_state
  -> substrate binding: HTTP, DB, file, queue, clock, random, IPC
```

The microservice envelope wraps this route with ingress and egress substrate
bindings. It does not replace or extend the route — it bounds it.

## Q1 — Service Request Envelope

**Evidence:** `ch7-runtime.md §7.3`, `ch12-effect-surface.md §12.2–12.3`,
`LAB-LANG-HTTP-TYPES-P1 (HttpRequest shape)`, `LAB-RACK-P14 (ContractResult branches)`

The service request envelope is the typed input structure that the substrate
binding parses and delivers to the RuntimeMachine before evaluation begins.
It carries all information required to: load the correct contract, verify
authority, set up capability passports, and preserve the audit trail.

### Shape

```
ServiceRequest {
  correlation_id:     String,          -- immutable, generated by ingress substrate
  contract_id:        String,          -- which contract to load from .igapp manifest
  effect_names:       [String],        -- declared effect_binding names expected to run
  input:              Map[String, Value], -- typed inputs, validated against input_ports
  authority_ref:      String,          -- who authorized this request (auth token ref)
  capability_passports: Map[String, CapabilityPassport],
                                       -- keyed by capability family: "storage", "file", etc.
  idempotency_key:    String | nil,    -- required for non-idempotent effects
  ingress_substrate:  String,          -- "http" | "queue" | "ipc" | "grpc"
  ingress_timestamp:  String,          -- ISO8601 from clock binding; not now()
  artifact_digest:    String,          -- sha256 of the loaded .igapp artifact_hash
  profile_ids:        [String]         -- profiles to activate; must be declared in contract
}
```

### Field Rules

| Field | Rule |
|---|---|
| `correlation_id` | Immutable; assigned by ingress; flows unchanged through all layers |
| `contract_id` | Must match a contract in the loaded manifest; refuse if absent |
| `effect_names` | Declared subset; cannot exceed the effect_bindings in the contract |
| `input` | Validated against `input_ports` before evaluate begins; type error = reject |
| `authority_ref` | Must satisfy `authority` clause of each declared effect; refuse if absent |
| `capability_passports` | One per required IO family; checked against `requirements.json` |
| `idempotency_key` | Required when effect is non-idempotent in a retry-enabled profile |
| `ingress_timestamp` | From the clock binding; never from `now()` (Covenant forbids ambient time) |
| `artifact_digest` | Must match the loaded manifest `artifact_hash`; refuse on mismatch |
| `profile_ids` | Only profiles declared in the contract's `profile` blocks are permitted |

### CapabilityPassport Shape (from LANG-IO-CAPABILITY-EXECUTOR-P1 / P1 Q5)

```
CapabilityPassport {
  passport_id:     String,    -- immutable; content-addressed
  family:          String,    -- "storage" | "file" | "network" | "queue"
  capability_id:   String,    -- matches declared capability name in effect contract
  authority_ref:   String,    -- who issued this passport
  issued_at:       String,    -- ISO8601 from clock binding
  expires_at:      String | nil,
  scope_ids:       [String],  -- which effect_binding names this passport covers
  profile_ids:     [String],  -- which profiles activated this passport
  revoked:         Bool
}
```

A passport is pre-negotiated before the request reaches RuntimeMachine evaluate.
The substrate binding is responsible for passport delivery; the CapabilityExecutor
is responsible for passport validation.

## Q2 — Response Envelope

**Evidence:** `ch12-effect-surface.md §12.3`, `Covenant Postulate 8 (Receipts Are Proof)`,
`Postulate 15 (Timeout Is Not Failure)`, `Postulate 26 (Audit Closure)`,
`LAB-STORAGE-CAPABILITY-P1 (QueryExecutionReceipt)`, `LAB-RACK-P14 (ContractResult branches)`

The response envelope is returned after all declared effects have been executed
and the output contract has been evaluated. It is always typed; it never carries
an unstructured exception.

### Shape

```
ServiceResponse {
  correlation_id:     String,          -- same as request; closes the audit chain
  contract_id:        String,          -- echoed from request
  kind:               String,          -- "ok" | "denied" | "effect_failure" | "type_error" | "runtime_refusal"
  output:             Map[String, Value] | nil,
                                       -- typed output from contract output_ports; nil on failure
  diagnostics:        [Diagnostic],    -- non-fatal warnings from evaluate; never suppressed
  receipts:           [EffectReceipt], -- one per executed effect attempt; always present
  effect_outcomes:    Map[String, String],
                                       -- keyed by effect_name; value = outcome kind
  response_timestamp: String,          -- ISO8601 from clock binding
  response_observation: ResponseObservation
                                       -- typed observation that closes the audit chain (P26)
}
```

### ResponseObservation Shape

```
ResponseObservation {
  observation_id:  String,             -- content-addressed
  kind:            "response_observation",
  correlation_id:  String,
  contract_id:     String,
  outcome_kind:    String,             -- mirrors ServiceResponse.kind
  receipt_refs:    [String],           -- receipt_ids of all emitted EffectReceipts
  evidence_digest: String,             -- sha256 of canonical receipts + output
  observed_at:     String              -- ISO8601 from clock binding
}
```

### EffectReceipt Shape (from P1 Q6)

```
EffectReceipt {
  receipt_id:           String,        -- immutable, content-addressed
  effect_name:          String,        -- from effect_binding declaration
  capability_id:        String,        -- from CapabilityPassport
  family:               String,        -- IO family
  authority_ref:        String,        -- who authorized this effect
  idempotency_key:      String | nil,
  idempotency_key_used: Bool,
  inputs_hash:          String,        -- sha256 of canonical inputs
  outcome:              String,        -- one of 7 EffectOutcome kinds
  substrate:            String,        -- "storage" | "file" | "http" | "queue" | ...
  emitted_at:           String,        -- ISO8601 from clock binding
  evidence_refs:        [String]       -- prior evidence this receipt builds on
}
```

### Outcome Kind Rules

| kind | Meaning | P15 / Covenant rule |
|---|---|---|
| `succeeded` | Effect completed; receipt is immutable proof | — |
| `failed` | Known error from external system | ObservedFailure |
| `partial` | Partial completion; reconciliation needed | — |
| `timed_out` | Time limit exceeded; outcome unknown | **P15: UnknownExternalOutcome, not ObservedFailure** |
| `unknown_external_state` | Request sent; no confirmation | **P15: UnknownExternalOutcome, not ObservedFailure** |
| `compensated` | Compensation triggered and completed | — |
| `cancelled` | Cancelled before start | — |

`timed_out` and `unknown_external_state` must never be treated as failure by
the response mapper. They require reconciliation, not retry without a new
idempotency key.

### Diagnostic Shape

```
Diagnostic {
  kind:    String,   -- "warning" | "info"
  code:    String,   -- structured code (e.g. "DIAG-W01")
  message: String,
  context: Map[String, String]
}
```

Diagnostics are non-fatal. They are always included in the response even on
success. A response with `kind: "ok"` may carry diagnostics.

## Q3 — Host / Substrate vs Igniter Semantics

**Evidence:** `LAB-IO-BOUNDARY-P1.md`, `ch12-effect-surface.md`, `LAB-RACK-P14 (authority statement)`

### The Boundary

| Layer | Owner | What it does |
|---|---|---|
| **Host** | External operator | Runs the accept-loop process; owns OS-level networking, signal handling, and TLS termination |
| **Substrate binding** | External code (Rack, AMQP client, gRPC stub, etc.) | Parses raw transport bytes to typed ServiceRequest; serializes ServiceResponse back to transport |
| **Igniter semantics** | `igniter-lang` runtime | Everything from ingress_observation onward: evaluate, effect dispatch, receipt production, audit chain |

### Rules

1. **Rack/HTTP is one substrate binding, not the architecture.** A microservice
   that uses HTTP/Rack differs from one that uses a queue only in how the
   substrate binding parses input and serializes output. The runtime evaluate
   path, CapabilityExecutor dispatch, and receipt production are identical.

2. **The host must not inject ambient state.** Environment variables, globals,
   thread-locals, and Rack `env` fields must not reach the RuntimeMachine
   evaluate layer. All inputs arrive as the typed `ServiceRequest.input` field.

3. **The substrate binding must not perform IO on behalf of the contract.**
   It may parse; it may validate input types; it may serialize output. It must
   not call the DB, file system, or network except through declared
   CapabilityExecutors.

4. **The semantic layer owns the contract identity check.** The substrate
   binding delivers `artifact_digest`; the semantic layer verifies it against
   the loaded manifest `artifact_hash`. This is a language boundary, not an
   operator concern.

5. **Rack env is not a SemanticIR node.** `IO.StorageCapability` is not an
   ActiveRecord connection string. The two layers must never exchange typed
   representations.

### Pipeline Diagram

```
[Host process: accept loop / queue poller / IPC daemon]
        |
        |  raw transport bytes
        v
[Substrate binding: Rack / AMQP / gRPC / IPC adapter]
        |
        |  ServiceRequest (typed)
        v
[Ingress gate]
   verify artifact_digest vs manifest
   validate input types vs input_ports
   check authority_ref present
   emit ingress_observation
        |
        v
[RuntimeMachine evaluate — pure]
   load contract from .igapp
   evaluate input_node + compute_node DAG
   build EffectPlan (ordered effect list with inputs)
   no IO here — deterministic
        |
        v
[CapabilityExecutor dispatch — for each effect in plan]
   verify passport present and valid
   verify authority_ref matches effect authority clause
   verify idempotency_key if required
   call registered executor for IO family
   collect EffectResult
   emit EffectReceipt
        |
        v
[Response construction — pure]
   map EffectResults to output contract
   attach receipts
   build ServiceResponse
   emit ResponseObservation (P26 audit closure)
        |
        v
[Substrate binding: serialize to transport]
   HTTP: typed status + headers + body
   queue: typed message body
   IPC: typed response record
        |
        v
[Host process: send response]
```

## Q4 — Allowlist: What Must Be Gated Before Effect Dispatch

**Evidence:** `ch7-runtime.md §7.8`, `ch12-effect-surface.md §12.5`,
`Covenant CR-001`, `LAB-STORAGE-CAPABILITY-P1 (6-gate pipeline)`,
`LAB-IO-BOUNDARY-P1 (fail-closed table)`

All gates are fail-closed. A missing or invalid value produces a structured
`RuntimeRefusal`, not an exception.

### Gate 1: Contract ID Allowlist

```
Rule: contract_id must exist in loaded .igapp manifest.contracts[]
Refusal: runtime_refusal / effect.unknown_contract
```

Only contracts that were compiled into the loaded artifact may be dispatched.
There is no dynamic contract loading at request time.

### Gate 2: Artifact Digest

```
Rule: ServiceRequest.artifact_digest must equal manifest.artifact_hash
Refusal: runtime_refusal / effect.artifact_digest_mismatch
```

Content-addressing ensures the contract being dispatched is exactly the one
that was compiled and audited. A mismatch indicates tampering or stale input.

### Gate 3: Capability ID Allowlist

```
Rule: each capability_passport.capability_id must match a declared capability
      in the effect contract's capability blocks
Refusal: runtime_refusal / effect.undeclared_capability
```

A passport for a capability not declared in the contract is silently invalid.
No executor may be called without a matching declaration.

### Gate 4: Profile ID Allowlist

```
Rule: each profile_id in ServiceRequest.profile_ids must be declared in the
      contract's profile blocks
Refusal: runtime_refusal / effect.undeclared_profile
```

Profiles that are not declared in the loaded contract cannot be activated at
request time. This prevents capability widening by environment or configuration.

### Gate 5: Authority Verification

```
Rule: ServiceRequest.authority_ref must satisfy the authority clause of each
      declared effect
Refusal: runtime_refusal / effect.authority_missing | effect.authority_mismatch
```

Effects marked `privileged` or `irreversible` in PROP-035 require an authority
ref. A request without one must be refused before evaluate begins.

### Gate 6: Capability Passport Validity

```
Rule: each CapabilityPassport must be:
      (a) not revoked
      (b) not expired (expires_at nil or in the future per clock binding)
      (c) scope_ids include the requested effect_name
Refusal: runtime_refusal / effect.passport_invalid | effect.passport_expired | effect.passport_scope_mismatch
```

### Gate 7: Idempotency Key

```
Rule: if the effect contract's profile is retry-enabled AND the effect is
      non-idempotent, idempotency_key must be present
Refusal: compile-time OOF-M4 (future PROP-035); runtime fallback: runtime_refusal / effect.idempotency_key_missing
```

### Gate 8: Executor Registration

```
Rule: a CapabilityExecutor must be registered for the declared IO family
Refusal: runtime_refusal / effect.unsupported_family
```

All eight gates must pass in sequence before any CapabilityExecutor is called.
No partial execution is permitted: if gate N fails, gates N+1..8 are not checked
for that effect.

## Q5 — Replay for a Request with Real IO

**Evidence:** `ch7-runtime.md §7.9`, `Covenant Postulate 8`, `Postulate 15`,
`LAB-STORAGE-CAPABILITY-P1 (receipt evidence-only invariant)`,
`LAB-SCHEDULING-RECEIPT-REPLAY-P1 (replay evidence)`

### Replay Invariant

Given the same `inputs_hash` + `idempotency_key` + `capability_id`, a
re-execution must not produce a duplicate external effect. The executor is
responsible for substrate-level deduplication; the language is responsible for
recording the key.

### What Must Be Recorded per Execution Attempt

| Field | Purpose |
|---|---|
| `effect_name` | Which declared effect ran |
| `capability_id` | Which capability passport was used |
| `inputs_hash` | sha256 of canonical inputs passed to executor |
| `outcome` | One of the 7 EffectOutcome kinds |
| `substrate` | Which substrate handled execution |
| `emitted_at` | Timestamp from clock binding |
| `idempotency_key` | For replay — same key = same logical operation |
| `idempotency_key_used` | Whether the key was presented to the executor |
| `authority_ref` | Who authorized the execution |

### Replay Flow

```
1. Caller retries with same idempotency_key
2. Ingress gate checks idempotency_key present in ServiceRequest
3. CapabilityExecutor receives (effect_name, passport, inputs, authority_ref, idempotency_key)
4. Executor looks up prior execution by idempotency_key at the substrate level
5. If found and outcome was `succeeded`: return original EffectResult without re-executing
6. If found and outcome was `unknown_external_state`: check substrate for current state
7. If not found: execute; record receipt
8. Returned ServiceResponse carries all receipts including replay-sourced ones
```

### Unknown External State

If the executor cannot determine whether the effect completed (network timeout,
queue unavailable, storage unreachable), it must return `unknown_external_state`,
not `failed`. The receipt records `last_known_ref`. Reconciliation is the
caller's obligation, not the runtime's.

**P15 is load-bearing:** the response mapper must not convert
`unknown_external_state` or `timed_out` to a failure branch in the output
contract. They are distinct outcomes that require explicit handling.

### Audit Chain Closure (Postulate 26)

Each request produces an evidence chain:

```
ingress_observation
  -> effect_receipt[0..N]
  -> response_observation
```

The `ResponseObservation.evidence_digest` is a sha256 of the canonical form
of all receipts and the typed output. This closes the audit loop for the
request. Postulate 26 requires that the decision is complete only when the
outcome feedback loop closes.

## Q6 — Rack-Shaped Prior Work: Reuse Boundary

**Evidence:** `LAB-RACK-P14 (authority statement)`, `LAB-LANG-HTTP-TYPES-P1 (HttpRequest/HttpResponse shapes)`,
`LAB-IO-BOUNDARY-P1`, `ch12-effect-surface.md`

### What Is Reusable as Typed HTTP Shapes

These surfaces from prior lab work are valid typed shape references for the
HTTP substrate binding:

| Surface | Source | Reuse scope |
|---|---|---|
| `HttpRequest { method, path, headers, body }` | LAB-LANG-HTTP-TYPES-P1 | Substrate binding input parsing |
| `HttpResponse { status, headers, body }` | LAB-LANG-HTTP-TYPES-P1 | Substrate binding output serialization |
| `ContractResult { kind, data_body, resp_headers }` branch taxonomy | LAB-RACK-P14 | ServiceResponse.kind mapping to HTTP status |
| 6-branch status map (200/201/404/403/502/503) | LAB-RACK-P14 | HTTP egress status selection |
| Two-gate dispatch (input validate + output validate) | LAB-LANG-HTTP-TYPES-P1 | Ingress/egress type gates |
| Failure taxonomy (failed / timed_out / unknown_external_state) | LAB-LANG-HTTP-TYPES-P1 | EffectReceipt outcome naming |

The substrate binding may use these shapes as typed record templates. They are
evidence that the shapes are sound; they are not authority for the Igniter
semantics layer.

### What Is Explicitly NOT Authority

| Surface | Source | Why not authority |
|---|---|---|
| `call_contract` | LAB-RACK-P1..P14 | Lab-only; not canon; not a stable surface |
| Rack `env` Hash | Rack gem | External schema; must not reach SemanticIR |
| Accept-loop / Rack middleware chain | LAB-RACK-P14 gap table | "Real Rack env / accept-loop: Closed (out of scope)" |
| `Igniter::ContractBuilder` / `GraphCompiler` | Old Ruby `igniter` gem | Not igniter-lang authority; explicitly closed |
| `map_get` VM bytecode | LAB-RACK-P14 gap table | Open gap; not available for use |
| Three-level chained field access | LAB-RACK-P14 gap table | Open gap; not available for use |
| ContractRef dispatch (lab-only) | LAB-LANG-HTTP-TYPES-P1 | Proof-local Ruby simulation; not canonical ContractRef |

### The Distinction

Rack-shaped prior lab work proved that:
- Typed HTTP record shapes are expressible and typecheck-sound.
- Branch taxonomies map correctly to HTTP status codes.
- Two-gate dispatch (input + output validation) is composable.

It did NOT prove that:
- Rack is the Igniter microservice architecture.
- `call_contract` is a stable dispatch surface.
- The old Ruby `igniter` gem and Rack gem are authority for `igniter-lang` IO.

The microservice envelope uses Rack/HTTP as one substrate binding. It does not
use Rack as the architectural foundation.
