Interactive Igniter game — click a body to kick it

Six bodies fall under integer physics; click one and it gets kicked up. The whole game is the Igniter app on igniter-vm — Step(world) (physics), View(world)→Scene (3D→2D projection), Reduce(world, id) (the click→kick) — all `.ig` contracts. The host only ticks, renders the projected scene, and hit-tests the click into a body id.

logic.ig Step (igniter-vm)
view.ig View → Scene (igniter-vm)
interaction.ig Reduce(world, id) (igniter-vm)

Running live here are the Rust mirrors of those three `.ig` contracts — proven bit-identical to them (cross-checked every run in examples/vm_game.rs, which drives the ACTUAL `.ig` on the VM). The browser uses the mirror only because the VM is native and rAF needs ~60fps; the logic, projection, and kick are exactly what the VM computes.
See the VM-frame playback game_ig and the Rust loop game.
